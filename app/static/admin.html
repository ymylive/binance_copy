ADMIN · Binance 登录

Binance 一键登录

Binance 跟单接口需要登录态 Cookie。这里提供「生成令牌 → 粘贴凭证 → 自动写入」三步式流程,无需手动 SCP 文件,也无需重启服务。粘贴的凭证会保存到服务器的 cookies.json,引擎会在 30 秒内识别并自动连入。

1
登录 Binance 跟单页
浏览器打开 https://www.binance.com/zh-CN/copy-trading 完成登录(账号要先看过任意一位带单员的详情页,后端才会下发跟单 Cookie)。
2
打开 DevTools 的 Network 面板
按 F12(macOS:⌥⌘I)→ 切到 Network 标签页 → 在筛选框输入 copy-trade → 在跟单页内点开任意一位带单员,Network 列表会立刻刷出新请求。
3
右键任选一条请求 → Copy → Copy as cURL (bash)
如果右键里没有 Copy as cURL (bash),选 Copy as cURL 也行;Firefox / Safari 菜单文案略有不同,但都在「复制」二级菜单下。
4
回到此页面 → 点「① 生成一次性令牌」→ 粘贴 cURL → 提交
提交后状态卡会自动变成 已连接,引擎会立刻使用新登录态轮询持仓,无需重启。
先点这里生成 10 分钟内有效的令牌,再粘 cURL
⬇ 把整段 cURL 粘到下面这个框里 (也接受 Cookie JSON 数组或原始 HTTP header 段;系统会自动识别)

为什么需要这一步? Binance 跟单接口对 HttpOnly Cookie + 浏览器签名做了严格校验,任何脚本/伪造请求都会被风控拦截。因此跟单引擎必须直接持有真实浏览器的登录态。这里的 cURL/Cookie 提取仅在你点击生成令牌后的 10 分钟内可用,且每个令牌只能使用一次。
